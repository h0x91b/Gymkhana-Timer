Added a small HUD pill showing the measured capture FPS and per-event timing precision (half the frame interval, in ms). Values come from rVFC metadata.mediaTime — the authoritative frame clock — averaged over a 30-frame rolling window and throttled to ~4 Hz so the readout stays legible. The pill is always visible once the camera is running (not hidden behind the Debug toggle) so the user can judge timing accuracy at a glance; localized in en/ru/es.

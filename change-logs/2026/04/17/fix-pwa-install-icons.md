Add real 192x192 and 512x512 PNG icons (helmet + stopwatch + cones artwork) to icons/. Without them, Android Chrome rejected installability and the service worker's cache.addAll failed on the missing files, blocking both the install prompt and offline mode. Also switched manifest.json orientation from landscape to portrait per updated UX preference. Added in-app update flow: app.js polls registration.update() on an interval and on tab focus, and when a new service worker is detected it either auto-applies (IDLE / FINISHED) or shows a terracotta "Update" button in the controls row (mid-run). sw.js no longer skipWaiting()s on install — clients drive the cutover via a SKIP_WAITING postMessage. Added runtime version badge (new version.js module): at startup fetches ./sw.js, extracts CACHE_VERSION, and formats its Last-Modified header into a tiny mono pill in the HUD, so which build is running is visible at a glance. Lowered default motion threshold from 0.30 to 0.25 (input default + Detector initial) to make detection a bit more sensitive. Bumped sw.js CACHE_VERSION gymkhana-v2 -> gymkhana-v7 so existing clients pick up every iteration.

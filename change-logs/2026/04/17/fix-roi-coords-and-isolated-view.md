Fixed ROI coordinate mapping: tap points from the overlay (CSS pixels) were being passed directly as source-rect arguments to drawImage, which expects the video's intrinsic pixel coordinate system. The conversion now accounts for the `object-fit: cover` scaling and centering offset, so the detector actually reads the region the user selected. Also added an isolated ROI view: after the user picks a region, the raw camera feed is hidden and a dedicated canvas renders only the cropped ROI (letterboxed to preserve aspect ratio), filling the stage. Bumped detector cooldown from 1.2 s to 3 s to prevent a single motorcycle pass from firing both start and finish. Added a HUD cooldown indicator — an Ivory pill with a terracotta progress bar that counts down the remaining debounce window, visible only during WAITING_START and RUNNING so users can see why a second trigger within the window is ignored.

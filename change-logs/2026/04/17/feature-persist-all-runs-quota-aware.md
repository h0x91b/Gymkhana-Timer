Storage.save now keeps run history open-ended: every finished run is appended and the full list is persisted to localStorage. Record shape expanded to { elapsed, startedAt, finishedAt } so the saved date reflects when the run actually began, not just when the finish line was crossed. On QuotaExceededError the store trims the oldest entries in 10% chunks and retries, with a hard floor of MIN_KEEP=100 most-recent runs before falling back to single-entry eviction. No UI changes yet — the expanded history is for future views.

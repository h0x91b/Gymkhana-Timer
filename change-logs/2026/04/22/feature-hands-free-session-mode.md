Shipped the hands-free session mode — the new core UX contract. As soon as the ROI is picked the app enters an automatic OBSERVING → ARMED → RUNNING → FINISHED → 10-second COOLDOWN → OBSERVING loop that the rider never has to interrupt between runs; no separate "Start session" tap. Auto re-arm waits for the ROI to be empty and still for 2 s before rebuilding the reference frame, which compensates for ambient-light drift and prevents tainted-reference false starts. Three saturated daylight-readable signal colours (grass-green "go", bright orange "wait", coral-red "error/come over") drive the whole body background so the rider can read state from 10 m under bright sun — no pulse animation, the colour alone is the signal. Voice keeps the rider in the loop even when they're not looking: "start" / "finish, N.N seconds" / "ready to go" on phase changes, plus a periodic "not ready" every 15 s while the session is active but not yet ARMED. The big route timer keeps the last run's value displayed through cooldown/observing/armed and only resets when the next run actually starts; a small sub-line renders the 10 s countdown, "Ready to go", or previous-run time depending on phase. Retired the old Arm / Cancel / New run button cycle in favour of a single implicit flow; Stop session is revealed by tap during hands-free. All "lap timer" wording replaced with "route timer" across README, manifest, package, TZ, AGENTS, DESIGN; i18n keys updated in en/ru/es. Added decision record 006.

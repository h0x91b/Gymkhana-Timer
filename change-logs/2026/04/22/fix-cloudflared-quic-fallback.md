Fix the cloudflared quick tunnel silently hanging after printing its trycloudflare.com URL. The default QUIC (UDP/7844) transport was being dropped by the current network, so cloudflared looped on `failed to dial to edge with quic: timeout: no recent network activity` while the local dev server looked healthy. The dev server now spawns cloudflared with `--protocol http2` by default (TCP/7844 works everywhere we tested), with an opt-in `CF_PROTOCOL=quic` env override for networks where UDP is fine. AGENTS.md documents the new env and the reasoning.
